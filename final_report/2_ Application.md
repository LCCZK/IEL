In my initial proposal I identified coding skill in different programming languages to be the only skill relevant to my SLICC. I managed to develope this skill when I had to read and understad source code done my other EUFS members. Reflecting on this experience, I put focous to improve my code quality by adopting industrial convensions and writing comments on my code so that future developer will have an easier time when they try to read my code. However, contradict to what I believed initially, the programming experience only form a small part of my SLICC. My role in EUFS as a team-lead requird me to carry out extra activities other than just coding, which involve different skill sets.

It was true that the first two years of my experience with EUFS were mostly just about coding, and I could rely on the senior members to test my code and provide feedbacks, since I became the senior member this year, I also have to take up such responsibility. To fulfill this taks I had to apply my software testing skills. Initially I manually run codes I recived on my machine, and sent screenshorts of the result as feedbacke to members, which then I recived complants made me realised that I was doing it too slow and screenshorts of a segmentation fault or unexpected outputs were not enough infomation for developers to fix their code. I then decided learn to set up CI/CD pipeline under the help of my peers, aimed to automate the testing process by performing tasks such as running build, unit test and intergration test command when a merge request is created, to generate feedback instanstly. I believe is a very useful skill to develope further, given it has cut the time I spent on testing other members' software by 90%, therefore I decided to enrol myself into the MS DevOp course during the summer.
